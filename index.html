1.1The Basics of the Web and HTML

The Internet

The Internet is a connection of computers that form a network that allows information to be distributed around the Globe.It is essentially made up of users sitting on their computers with normal screens and mice, and servers, computers that are designed to serve files quickly. The user sends out a request to the sever using HTTP "Hyper Text Transfer Protocol". The server then sends back a file which is generally a HTML "Hyper Text Markup Language" document.

HTML - Hyper Text Markup Language

HTML documents form the basis of the Internet as we see it on our browsers. They contain text in a language that the browser can convert into the webpages we are familiar with. This text uses Tags which can do simple things, like make text BOLD or Italicized. Or it can insert pictures, or links to other places - Such as this link to Wikipedia.

HTML documents are also broken down in sections which contain different sorts of information. There is the document type(these days generally just "HTML"), the title (The description in the browser title bar) , the header (containing CSS elements and other info), and the body (which is where the main content goes. As explained in this video.

In-line Vs Block

There are 2 basic sorts of Tags (or elements). These are in-line and Block

As the name suggests, In-line, means that the tag doesn't interrupt the line. The text would continue to flow. Bold, Italics, images and links are examples of In-line tags.

Block tags on the other hand create blocks of text within them. These paragraphs are written in the block p tags. Div, header and body are also block tags.

You Don't Need to Know Everything

Help is just a quick Google search away.

Can't remember how to make text bold? just Google, "HTML make text bold" and the answer will quickly become apparent.

Just focus on learning the higher level information and a deeper understanding of how it all fits together and then you can Google the exact syntax of a language if you forget.

1.2 Creating a Structured Document

How Do Websites Work?

In general a website is made up of three basic elements and we can think of these with the analogy of a house.

  • First there is the structure, the walls and rooms of a house or the HTML on a webpage.
  • Then we have the styling, the colour of the paint, curtains etc. This corresponds to the CSS elements.
  • Lastly we have the interactive components, your TV, phone, lights etc, which can all be turned off and on or changed in real time. This is Javascript.

In order to build our house or website we have rules to follow and tools that we can use.

Tools

There are some great tools that we can use as we learn to be programmers.

For learning front end web development we have Scratchpad, Codepen, and Sublime Text

A really good video describing the differences can be found here...

Another great tool to use as you learn front end web development is to see what others have done. You can do this easily on Chrome. If you find a webpage that you like, you can right click on an element and select "inspect element" and it will bring up the DOM of the page where you can see the HTML and CSS components and how they interact.

You can also play with the values in this area and the page will update in real time. NB. changes aren't permanent and when you refresh the page you'll loose whatever you did. But it's a great way to explore how values effect different elements.

Tree Like Structure

Web pages can be broken down into a tree like structure. For more information on Tree Structures visit this Wikipedia page.

This means that we can follow sections of the web page down in blocks. For example you could have a Div block which contains multiple paragraph blocks some of which could contain in-line tags, such as links or images. Just like this page you're reading now.

HTML, CSS & DOM

In terms of structure and style we can brake the website down into HMTL - Hyper Text Markup Language, CSS - Cascading Style Sheets and DOM - Document Object Model.

Both HTML and CSS and languages. HTML is a language of structure and CSS is a language of style. They each have there own specific syntax but work synergistically. HTML is made up of Tags which contain Content. The Tags can have classes and attributes which are referenced in the CSS and those values provide the styling for that tag.

DOM is not a language, it is essentially a way of describing how the browser breaks down the HTML and CSS into the tree structure that is displayed on the page.

1.3 Adding CSS Style and HTML Structure

CSS - Cascading Style Sheets

One of the biggest ideas in CSS is that of selectors.

Selectors are a way of describing which parts of the HTML the styling should apply to. There are several different types of selectors which include but are not limited to:

  • Class Selectors - these are used in elements like divs where you could have div class = "hello" and in the style sheet you would describe styling that apply to any element with the class "hello"
  • Tag Selectors - This is for more general elements in the HTML document like p (paragraph), h2 (heading) and links.

You can also blend styles for elements that belong to multiple tags and classes. eg. "h2" elements inside blocks with class "hello".

Again it is not important to learn all selectors now, but rather to know where to look. A great reference can be found here.

The really key thing to remember about CSS though is that a more specific definition will override a less specific one. One way to think of this is like driving on the open road. All highways in New Zealand have a set speed limit of 100km/hr. But sometime you might come across local road works and the speed drops to 30km/hr. The local specific definition of the speed in the work zone overrides the more general rule for highways.

The Box Model

Every element in the HTML document is contained in a box and is subject to the Box Model. You can play around with this here...

Every rectangle or box on the page can be broken down into 4 sections, the size of which can be defined as either a pixel or percentage. Percentages are better for changing screen sizes, which is pretty standard these days.

  • Content - Found at the centre this is where the actual content lives, that could come in the form of text, and image, video etc.
  • Padding - This creates a space around the content but is subject to the background of the box.
  • Border - This is a physical / visible border around the content and is also set to the boxes background unless specified.
  • Margin - This is more of an invisible boundary around the border. It creates a space between the border and other items found around it.

By default the element / box size is the sum of all the widths and heights of the inner 3 sections (content, padding, border) of the element. Because of this everytime we alter a value the size of the box would change. This makes keeping all the elements in line a difficult task.

Thankfully there is a css selector we can use called "box-sizing: border-box". When we use this we can play with the padding, border and content size and the box size will stay the same. Although this method does not include the margin. You do have to be aware though that this wont work on all (older) browsers.

CSS & Repetition

One of the main reasons that CSS was created and that we use it is to avoid repetition. There are 2 benefits from this:

  1. Less code to write- By storing styles in CSS we can refer to the same code multiple times with just a few characters, thus reducing the amount of code we have to write. This is nice in itself, but an added benefit is that this also reduces mistakes. Everytime you have to write a piece of code there is opportunity to get a character wrong. This can have large unintended result. CSS dramatically reduces this.
  2. Consistent design - By having the styles stored in one place it means that we can easily apply consistent styling to similar elements on a page. Take these lesson ideas for example. By having the idea contained in a div with the class="lesson-idea" and then styling for that class in the CSS document it means that all ideas have the same styling. This would be difficult to remember or achieve if we tried typing it each time. Even using cut and paste can have unexpected consequences. Also this means that we can change the style for all idea elements quickly to test new designs rather than altering each one individually.

Code, Test, Refine

When working through any project, but in this case building the code to display a webpage as provided by a designer, you can follow a simple 3 step process

Code, Test, Refine and repeat

The way that you pick which elements to work on first will alter depending on the type of project. But for this style of web design a simple system would be:

  1. Break up into boxes
  2. Identify repeated styles / elements
  3. Start writing your HTML
  4. Start writing your CSS (from biggest to smallest)
  5. Identify the small errors and fix them
